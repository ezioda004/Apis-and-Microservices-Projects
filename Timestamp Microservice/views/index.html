Timestamp Microservice

User Stories:

  1. The API endpoint is localhost:1337/api/timestamp/:date_string?
  2. A date string if valid can be successfully parsed.
  3. If the date string is empty, the service uses the current timestamp.
  4. If the date string is valid the api returns a JSON having the structure
  5. If the date string is invalid the api returns a JSON having the structure {"error" : "Invalid Date" }

Example Usage:

localhost:1337/api/timestamp/2015-12-25 localhost:1337/api/timestamp/1450137600000 http://localhost:1337/api/timestamp/10 10 1000

Example Output:

{"unix":1451001600000, "utc":"Fri, 25 Dec 2015 00:00:00 GMT"}
